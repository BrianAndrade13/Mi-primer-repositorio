Guía práctica: Creación de cuenta y
repositorio público en GitHub
Unidad: 4 – Control de versiones
Duración estimada: 30 a 45 minutos
Modalidad: Práctico Experimental (APE)

Objetivo de la práctica
Aprender a:
1. Crear una cuenta personal en GitHub.
2. Configurar un repositorio público.
3. Organizar una estructura de carpetas y archivos (.csv) dentro del repositorio.

Requisitos previos
- Conexión a internet.
- Correo electrónico activo.
- Navegador actualizado (Chrome, Edge o Firefox).

Parte 1: Crear una cuenta en GitHub
1. Ingresa a la página principal: https://github.com
2. Haz clic en 'Sign up' (registrarse).
3. Ingresa tu correo electrónico y haz clic en 'Continue'.
4. Crea una contraseña segura y sigue las instrucciones.
5. Escribe un nombre de usuario.
6. Verifica que no eres un robot.
7. Revisa tu correo electrónico y confirma la cuenta con el enlace que te envía GitHub.
8. Inicia sesión desde https://github.com/login.

Parte 2: Crear un repositorio público
9. Haz clic en el ícono '+' y selecciona 'New repository'.
10. Completa los campos: nombre, descripción y selecciona la opción Public.
11. Marca la casilla 'Add a README file'.
12. Haz clic en 'Create repository' para crear el repositorio.

Parte 3: Crear una estructura de carpetas y archivos
Estructura solicitada:
mi-primer-repositorio/
│
├── datos_ventas/
│ └── ventas.csv
│
└── datos_clientes/
└── clientes.csv
13. Dentro del repositorio, haz clic en 'Add file → Create new βile'.
14. Escribe el nombre 'datos_ventas/ventas.csv'.
15. Agrega contenido CSV de ejemplo y guarda el archivo.
16. Repite el proceso para crear 'datos_clientes/clientes.csv' con datos de ejemplo.

Parte 4: Verificación final
1. Verifica que las carpetas y archivos sean visibles en el repositorio.
2. Abre los archivos .csv para confirmar que se muestran correctamente.
3. Copia la URL pública del repositorio para entregar en un documento .PDF
4. # Mi-primer-repositorio
